⚙️ 基本设定

1. 规则描述

  • 采购系统中，采购申请 (PR) 要批准后才能生成采购订单 (PO)，0 表示审批通过，1 表示审批拒绝，2 表示待审批。
  • 单价高于 100 元的商品，每次申请单独生成一条 PO；单价相同且小于等于 100 元的相同商品合并到一条 PO。
  • 商品单价小于 100 元且数量大于等于 100 的商品单价打九折。
  • 按商品 ID 升序排序，ID 相同则按数量降序排列。

2. 输入描述

  • 第一行输入 PR 的数量 n。
  • 接下来 n 行，每行输入四个整数表示商品 ID、数量、单价、PR 状态。

3. 输出描述

  • 商品 ID、数量、单价。

🔎 重点解析

1. 整体逻辑

搞清楚每轮输入是采购申请 PR，最终生成是采购订单 PO。

  1. 定义采购订单类 POItem，一条订单对应一个实例对象。
  2. 读取并且按状态过滤 PR
  3. 按 100 元的界限，给 PR 分类，存储到不同结构里。对于价格小于等于 100 元的商品，
  4. 应用折扣
  5. 合并 PO
  6. 排序输出

2. 对象排序

  1. Comparable 接口
class POItem implements Comparable<POItem> {...}

让订单类实现Comparable (lang) 接口，必须重写 compareTo() 方法，自定义排序的规则；<POItem>泛型表示当前类只能和POItem类比较。 2. compareTo 方法

public int compareTo(POItem other) {
        if (this.itemID != other.itemID) {
            return Integer.compare(this.itemID, other.itemID);
        }
        return Integer.compare(other.quantity, this.quantity);
    }
  • compareTo()传入一个对象与当前对象相比较，只需明白：compareTo()返回负值时，当前对象在前；返回正值时，当前对象在后。
  • 一般使用Integer.compare()方法来操作compareTo()的返回值。Integer.compare()传入两个整型，若左边小于右边，返回负值；大于则返回正值，等于返回零。所以把当前对象放在左是升序，放在右则是降序。
  1. sort 方法
Collections.sort(allPOs);
  • Collections (util)类的sort()方法会按照compareTo()完成排序。
  • Collections是一个只有static方法的工具类，用来操作集合，常用方法有sort()、reverse()、shuffle()、max()。

3. getOrDefault 方法

  1. Map 接口的getOrDefault()方法：
int[] currentData = lowPriceProducts.getOrDefault(itemID, new int[]{0, price});
// V getOrDefault(Object key, V defaultValue)

在对应的 Map 中查找键为key的值，若有，返回对应的V；若没有，返回指定的defaultValue。 本题则是查找 PO 中是否有相同itemID，若有则把此次 PR 并入 PO，若没有就新建一个 PO。 2. 合并

currentData[0] += quantity;
lowPriceProducts.put(itemID, currentData);

Map 的 V 是一个包含两个整型的数组，第一个表示数量，第二个表示价格。所以先把原来的取出来传给临时数组currentData[]，currentData[0]第一位加上此次 PR 的数量就是累计数量。最后用put()覆盖原来的记录。

4. Java 集合

  1. 框架简单图解
Collection                     Map
├── List                       ├── HashMap
│   ├── ArrayList              │   └── LinkedHashMap
│   └── Vector                 ├── TreeMap
└── Set                        └── Hashtable
    ├── HashSet
    └── TreeSet
  1. 遍历解析
for (Map.Entry<Integer, int[]> entry : lowPriceProducts.entrySet())
  • Map 不可以直接遍历，要先使用 entrySet() 转换为包含键值对的 Set集合，也就是说lowPriceProducts.entrySet()这个整体表示集合，集合类型是Set<Map.Entry<Integer, int[]>>；集合里的元素是Map.Entry键值对，用<Integer, int[]>规定键值对的格式。所以最终返回给entry的类型是Map.Entry<Integer, int[]>，表示一个键值对。
  • Entry理解为表的一行，一个条目。
int itemID = entry.getKey();
int totalQuantity = entry.getValue()[0];
int originalPrice = entry.getValue()[1];
  • 使用getKey()获取键，getValue()获取值。

5. 数学函数

  1. 向上取整Math.ceil()
finalPrice = Math.ceil(originalPrice * 0.9);
  1. 向下取整Math.floor()
  2. 四舍五入Math.round()

💻 代码实现

import java.util.*;


// 采购订单 PO
class POItem implements Comparable<POItem> {
    int itemID;
    int quantity;
    int price;

    public POItem(int itemID, int quantity, int price) {
        this.itemID = itemID;
        this.quantity = quantity;
        this.price = price;
    }

    // 重写 compareTo 方法自定义排序规则
    @Override  // 告诉编译器这是一个重写方法
    public int compareTo(POItem other) {
        if (this.itemID != other.itemID) {
            return Integer.compare(this.itemID, other.itemID);
        }
        return Integer.compare(other.quantity, this.quantity);
    }
}


// 处理采购申请 PR 生成采购订单 PO
public class PurchaseOrders {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        // ---  处理 PR ---

        // 读取 PR 总数
        int n = scanner.nextInt();

        // highPricePOs 列表存储单价高于 100 的商品
        List<POItem> highPricePOs = new ArrayList<>();
        // lowPriceProducts 字典存储单价低于或等于 100 的商品
        // K 是商品 ID, V 是一个包含数量和价格的临时对象数组
        Map<Integer, int[]> lowPriceProducts = new HashMap<>();

        // 每轮循环处理一条 PR
        for (int i = 0; i < n; i++) {
            int itemID = scanner.nextInt();
            int quantity = scanner.nextInt();
            int price = scanner.nextInt();
            int status = scanner.nextInt();

            // 规则 1：只处理通过审批 (status == 0) 的 PR
            if (status != 0) {
                continue;
            }

            // 规则 2：根据单价将商品分类
            if (price > 100) {
                // 单价高于 100 元的商品，直接添加到列表
                highPricePOs.add(new POItem(itemID, quantity, price));
            } else {
                // 合并单价低于或等于 100 元的商品
                // 先找是否有当前商品 ID 的记录
                int[] currentData = lowPriceProducts.getOrDefault(itemID, new int[]{0, price});
                // 注：若同一商品前后价格不一致，会按首次输入为准
                currentData[0] += quantity;
                lowPriceProducts.put(itemID, currentData);
            }
        }

        scanner.close();

        // --- 生成 PO ---

        // allPOs 列表用于存储所有最终的 PO
        List<POItem> allPOs = new ArrayList<>(highPricePOs);

        // 规则 3: 打折
        for (Map.Entry<Integer, int[]> entry : lowPriceProducts.entrySet()) {
            int itemID = entry.getKey();
            int totalQuantity = entry.getValue()[0];
            int originalPrice = entry.getValue()[1];
            double finalPrice = originalPrice;

            if (originalPrice < 100 && totalQuantity >= 100) {
                finalPrice = Math.ceil(originalPrice * 0.9);  // 向上取整
            }
            allPOs.add(new POItem(itemID, totalQuantity, (int)finalPrice));
        }

        // 规则 4: 对 PO 排序
        Collections.sort(allPOs);

        // 输出最终结果
        for (POItem po : allPOs) {
            System.out.println(po.itemID + " " + po.quantity + " " + po.price);
        }
    }
}
