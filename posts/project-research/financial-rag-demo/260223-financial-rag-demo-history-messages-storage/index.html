🧬 继承BaseChatMessageHistory

1. 必须实现的核心方法

  • message属性：获取历史消息列表。
  • add_message()方法：添加新消息。

2. 意义

附加历史消息进链时，RunnableWithMessageHistory期望一个BaseChatMessageHistory类型的对象，继承此类可以被框架识别，内部调用标准方法。

🚀 初始化 __init__(self, session_id, storage_path)

1. 初始化存储文件路径

self.file_path = os.path.join(self.storage_path, self.session_id)

对象初始化接收存储路径和用户 id，用os模块的os.path.join()方法将传入的存储路径和用户 id 拼接成一个 JSON 文件的完整路径

2. 工厂函数指定文件路径

def get_history(session_id):
    return FileChatMessageHistory(session_id, "./chat_history")

💬 消息属性messages(self) -> list[BaseMessage]

1. @property装饰器

@property
def messages(self) -> list[BaseMessage]:
  • @property把messages()转换成属性，调用时像访问属性一样，因为langchain内部如RunnableWithMessageHistory等组件期望一个属性。
  • 懒加载：
    • 调用时才会触发文件读取，节省资源；
    • 动态更新，文件中途被修改会重新读取，非懒加载只会停留在初始化阶段。

2. try-except异常处理

  1. 异常：程序运行时发生的错误或意外情况，本质是抛出一个实例对象。
try:
    with open(self.file_path, "r", encoding="utf-8") as f:
        message_data = json.load(f)
        return messages_from_dict(message_data)
except FileNotFoundError:
    print("📁 文件不存在")
    return []
except json.JSONDecodeError as e:
    print(f"📄 JSON 格式错误：{e.msg}，位置：{e.pos}")
    return []
  1. 流程：先尝试执行try里的代码；如果抛出异常，匹配except后的异常，执行对应的代码；没有匹配到的异常程序才会崩溃。
  2. 注意：模块异常需要模块前缀引用。
  3. 额外：e.msg是错误详细信息；e.pos是错误位置。

3. 数据处理

  1. json.load()：json模块的方法，从文件中读取 JSON 格式的数据，转换成 Python 对象，一般是包含多条字典的列表。如：
[ {'type': 'human', 'data': {'content': '你好'}},
  {'type': 'ai', 'data': {'content': '有什么可以帮你的？'}}]
  1. messages_from_dict()：LangChain 的工具函数，把一个元素是包含特定 K 的字典的序列转换成 BaseMessage 消息对象列表。接上个例子：
[ HumanMessage(content='你好'),
  AIMessage(content='有什么可以帮你的？')]

📥 添加消息add_messages(self, messages: Sequence[BaseMessage])

1. 获取原消息列表

直接访问 (调用)self.messages属性获得一个list[BaseMessage]实例。

2. 添加消息

使用extend()将一个可迭代对象添加到原列表后面。

3. 转换字典

new_messages = [message_to_dict(message) for message in all_messages]

message_to_dict：把 一个 BaseMessage 消息对象转换回包含特定 K 的字典。配合列表推导式构建列表。

🔀 数据流动总览

  1. 历史消息以 JSON 格式存储，可看作包含字典的列表。
  2. 新对话是以Sequence[BaseMessage]的形式传入模块，原有的历史消息是以list[BaseMessage]的形式读入，两者附加成一个新的list[BaseMessage]型列表。
  3. 把新列表里的BaseMessage对象逐个取出转换成若干{'type': 'xxx', 'data': 'xxx'}形式的字典。
  4. 把字典组成一个列表，可看作 JSON 格式。
  5. 重新写回 JSON 文件。
