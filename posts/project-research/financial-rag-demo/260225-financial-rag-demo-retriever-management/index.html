🚀 初始化__init__(self, embedding)

1. 接收一个嵌入模型类

self.embedding = embedding

2. 初始化向量数据库连接信息

self.vector_store = Chroma(
        collection_name=config.collection_name,
        embedding_function=self.embedding,
        persist_directory=config.persist_directory
    )

与KnowledgeBaseService相同，依旧是三个参数：

  • collection_name：数据库的名称。
  • embedding_function：向量转换使用的模型。
  • persist_directory：数据库的位置，持久化地址。

🔄 把数据库转换为检索器

1. as_retriever()方法

def get_retriever(self):
    return self.vector_store.as_retriever(search_kwargs={"k": 5})

vector_store是langchain_chroma模块下的Chroma对象实例，as_retriever() 把数据库对象转换为检索器对象 (VectorStoreRetriever)，也就是说RetrieverConnectionService模块中的get_retriever()方法返回的检索器，对应一个指定的数据库，只会从此数据库中检索文本。返回的对象可直接插入链路中。

2. search_kwargs

  1. 定义：LangChain 向量存储检索器的搜索参数字典，用于配置检索行为。
  2. 原理：调用as_retriever()方法时，search_kwargs被传给底层的例如similarity_search()等方法，根据参数检索。
  3. 常用标准参数：
键 类型 说明 示例
k int 返回的最相似文档数量 (Top-K) {"k": 5}
score_threshold float 相似度分数阈值 (0-1)，只有高于此阈值的文档才会被返回 {"score_threshold": 0.7}
filter dict 元数据过滤器，根据文档的metadata进行筛选 {"filter": {"source": "report.pdf", "year": 2026}}
fetch_k int 初始获取的候选文本数量 (用于后续过滤) {"fetch_k": 20}
lambda_mult float MMR 多样性参数 (0-1)，越大越多样化 {"lambda_mult": 0.5}
4. 最大边界相关检索 (Maximal Marginal Relevance)
  • 默认使用相似度检索，添加search_type参数后按指定的检索类型：
def get_retriever(self):
    return self.vector_store.as_retriever(
        search_type="mmr",  # MMR 方式
        search_kwargs={
            "fetch_k": 10,
            "lambda_mult": 0.5,
            "k": 5
        }
    )
  • 流程解析：
    1. 告诉系统用 MMR 方式检索；
    2. 先按照相似度返回候选文本的数量；
    3. 指定多样性参数；
    4. 最终返回的结果数。
  • 优势：降低重复性，返回多角度的信息。
