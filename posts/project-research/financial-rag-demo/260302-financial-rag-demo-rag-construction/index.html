📜 历史记录附加

1. 用户原始输入

res_stream = st.session_state["rag"].chain.stream({"input": prompt}, {"configurable": {"session_id": "user_001.json"})
  1. 参数：
    • propmt：用户原始输入的字符串；
    • st.session_state["rag"]：RagService对象实例；
    • chain：rag.py中的RunnableWithMessageHistory类型属性；
    • stream()：LangChain 的Runnable接口的核心方法，返回生成器，用于流式传输出。
  2. 行为：这两个字典会先被传递给rag.py中的conversation_chain 添加历史消息。

2. 添加历史消息

conversation_chain = RunnableWithMessageHistory(
        chain,                          # RAG 流程链路
        get_history,                    # file_history_store.py 模块中定义的方法，期望一个 session_id
        input_messages_key="input",     # 告诉 RunnableWithMessageHistory 哪个是用户原始输入
        history_messages_key="history"  # 告诉 RunnableWithMessageHistory 把历史记录添加到哪个键
    )

流程：

  1. RunnableWithMessageHistory会自动从字典{"configurable": {"session_id": "user_001.json"}中取出"user_001.json"作为这个参数传入，返回一个FileChatMessageHistory对象。
  2. 对于这个FileChatMessageHistory对象，RunnableWithMessageHistory做了两件事：
    • 访问messages属性，返回之前的历史消息列表，格式为list[BaseMessage]：[HumanMessage(...), AIMessage(...), ...]。
    • 从字典{"input": prompt}提取出prompt转换成list[BaseMessage]格式传入add_messages()方法，把此次输入也永久化。
  3. 创建一个键值对，键是history_messages_key，值是历史消息列表，添加到原先的{"input": prompt}字典中，最终变成{"input": "prompt", "history": [HumanMessage(...), AIMessage(...)]}的形式。
  4. 把这个字典传入chain，开始了 RAG 流程。

🔎 增强深度检索

1. 必要前置知识

  1. 要明确chain的初始数据形式：{"input": "prompt", "history": [HumanMessage(...), AIMessage(...), ...]}。
  2. RunnableLambda()把普通函包装成 LangChain 的 Runnable 组件，使其可以参与链式调用。

2. 检索链

"context": RunnableLambda(format_for_retriever) | retriever | RunnableLambda(format_document)
  1. 提取出用户原始输入：format_for_retriever()返回input的值作为字符串传给检索器retriever。
  2. 深度检索：retriever是VectorStoreService类型的对象，实现了Runnable接口，通过invoke()接收字符串并进行向量转换、检索文本。最终返回一个文件列表list[Document]。Document是标准类，内部有两个字符串属性：page_content是内容，metadata是元数据。
  3. 规格化文本：format_document()把所有文件的内容和元数据拼接成一个字符串，赋值给"context"键。

3. 初始化检索得到的数据

链路最开始的字典形式：

{
    "input": {
	            "input": "prompt",
                "history": [HumanMessage(...), AIMessage(...), ...]
             },
    "context": "..."
}

🤖 模型总结并生成回复

1. 构建提示词模板

| RunnableLambda(format_for_prompt_template) | self.prompt_template
  1. 初始化适用于提示词模板的字典：format_for_prompt_template()用简单的[]取值提取出键为 input、history、context 三个键值对，返回一个字典，于是原先的字典变成了：
{
	"input": "prompt",                                   # 字符串
	"history": [HumanMessage(...), AIMessage(...), ...]  # 列表
	"context": "..."                                     # 字符串
}
  1. 构建提示词模板：
self.prompt_template = ChatPromptTemplate.from_messages(
	[
		("system", "你是一位专业的理财投资顾问...参考资料：{context}。"),
		MessagesPlaceholder("history"),
		("user", "请回答用户提问：{input}")
	]
)
  • from_messages()期望一个元组列表，内置几个标准值：
    • system：AI 的背景设定，把检索的知识提前灌输给 AI，达到一个“懂”的假象，这就是 RAG 的核心思想。另外可以填入角色设定等一系列要求。
    • user：用户最新的最原始的输入。
    • history：历史消息。
  • MessagesPlaceholder("history")：把历史消息列表展开，用于生成最终消息列表。提示词模板最终给模型的是一个list[BaseMessage]。

2. 获取回复

| self.chat_model | StrOutputParser()
  1. 模型接收的列表形式：
[
    SystemMessage(content="你是一位专业的理财投资顾问...参考资料：文档片段：xxx\n文档元数据：yyy"),
    HumanMessage(content="上一个问题"),
    AIMessage(content="上一个回答"),
    HumanMessage(content="请回答用户提问：用户当前问题")
]

返回一个AIMessage对象：

AIMessage(
    content="回答内容",           # 实际文本
    response_metadata={          # 元数据
        'token_usage': {...},    # token 使用情况
        'model_name': 'qwen',    # 模型名称
        'finish_reason': 'stop'  # 结束原因
    },
    id='run-xxx',                # 请求 ID (用于追踪)
    usage_metadata={...}         # 使用量统计
)
  1. 提取文本：StrOutputParser()从AIMessage对象中提取content返回字符串。

💬 输出最终内容

1. 生成器对象

res_stream = st.session_state["rag"].chain.stream({"input": prompt}, config.session_config)

res_stream是一个生成器对象，类型大致是Generator[str, None, None]。

2. 生成器函数

def capture(generator: Generator[str, None, None], cache_list: list):
    for chunk in generator:
        cache_list.append(chunk)
        yield chunk
  1. yield关键字让capture()变成一个生成器函数，返回生成器对象。
  2. capture()做了两件事：
    • 把每一次的回复保存到一个列表，用st.session_state["message"]记录到 Streamlit 的界面历史记录，达到一个聊天界面的效果；
    • yield chunk流式返回数据，配合write_stream()，达到一个打字机的效果。
  3. 事实上capture()从一个生成器对象流式获取值，再流式返回的生成器对象。

3. Streamlit

st.chat_message("assistant").write_stream(capture(res_stream, ai_res_list))

write_stream()接收生成器对象，实现流式输出效果。"assistant"表示这是 AI 说的话，输出在页面左边，头像是一个机器人；相对应的"user"表示用户说的话，输出在页面右边，头像是一个人。这是 Streamlit 内置好的功能，简单实用。
