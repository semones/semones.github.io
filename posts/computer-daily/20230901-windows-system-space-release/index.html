Windows10/11 系统空间释放建议

本文会讲解几种 Windows 完全原生的方法来释放 C 盘空间，注意每种方法都存在一定程度风险，仅供参考。

1. 删除系统恢复分区

系统恢复分区是安装新系统时自动分配的隐藏磁盘分区，可以在磁盘管理工具查看但无法进行管理操作。考虑到实际需要系统恢复时往往使用第三方工具，建议关闭。以管理员身份运行 PowerShell，依次输入：

  1. diskpart 用 PowerShell 操作 DiskPart 工具
  2. list disk 列出所有磁盘，例如笔记本一般只有一块 SSD 硬盘，磁盘号就是 0
  3. select disk 0 选择 0 号磁盘
  4. list partition 列出磁盘 0 的所有分区，可以在类型一栏看到恢复分区的分区号，假设是 3
  5. select partition 3 选择 3 号分区
  6. delete partition override 最重要的一步，删除所选分区，成功后会有提示

成功删除后 Win + X，再按 K 打开磁盘管理工具，可以看到原来的恢复分区已经变成未分配的空间，右键单击左邻的分区选择“扩展卷”即可合并。

2. 关闭休眠

休眠文件 hiberfil.sys 隐藏在 C 盘根目录，一般几个 GB 大小，如果不用休眠功能可以关闭来释放这部分空间。以管理员身份运行 PowerShell，输入：

  • powercfg /h off 关闭休眠功能
  • powercfg /h on 开启休眠功能

3. 减少虚拟内存

虚拟内存文件 pagefile.sys 隐藏在 C 盘根目录，一般几个 GB 大小，普通用户基本不会用到这个功能，建议修改大小，步骤如下：

  1. 依次选择高级系统设置 → 性能设置 → 高级 → 更改虚拟内存。
  2. 取消勾选“自动管理所有驱动器的分页文件大小”，点击 C 盘选择“自定义大小”，将初始大小设置为系统的最低要求 (与设备主存大小相关)，最大值可视情况填写。

4. 关闭系统保留空间

以管理员身份运行 PowerShell，输入：

  1. DISM.exe /Online /Get-ReservedStorageState 查看是否开启
  2. DISM.exe /Online /Set-ReservedStorageState /State:Disabled 禁用

5. 关闭系统还原点

依次执行：运行 gpedit.msc → 计算机配置 → 管理模块 → 系统 → 系统还原 → 关闭系统还原 → 启用

6. 关闭传递优化

传递优化是类似 BT 下载的 P2P 技术，只不过用于下载系统更新，现在网络速度很快，作用不大。打开系统设置：

  1. 在“更新和安全”中关闭传递优化
  2. 在“系统“的”存储”中删除传递优化缓存

7. 删除系统更新残留 (仅供参考，谨慎采纳)

Windows 系统每次更新会在C:\Windows\servicing\LCU目录留下大量文件。LCU 即为 Latest Cumulative Update，最新累积更新，删除后可能会导致：

  1. 系统更新失败或出错
  2. 无法回滚到之前的更新版本
  3. 系统修复功能失效
  4. 系统不稳定

由于我从不用系统自带的各种恢复功能，所以每次更新完成后都会删除掉，一直正常使用，无任何影响。此项仅供参考不推荐删除，如产生任何问题概不负责。
