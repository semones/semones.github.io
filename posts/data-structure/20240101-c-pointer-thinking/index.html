对于 C 语言指针的理解与想法

Semones included in 数据结构

Contents

    • 2. 指针

理解一门语言要从底层入手。

1. 地址

C 语言的编译器会把.c文件编译为机器码存入一个.exe文件，机器码可以对应汇编语言，是直接与 CPU 对话的语言。各类操作系统比如 Windows、Linux 会提供 API 支持 C 程序的执行，可以看作一个巨大的“运行环境”。对比 Java 的编译器会把.java文件编译为字节码存入一个.class文件，其实思路是相似的，字节码是与 JVM 对话的语言，JVM 就是一个虚拟机，模拟了操作系统和 CPU，它需要把字节码翻译成各种机器码与不同架构的 CPU 对话。JVM 是运行在系统上的，而 Java 程序是运行在 JVM 上的。

进程直接使用物理地址会引发冲突，所以需要隔离，由此引入虚拟内存技术。进程管理的地址都是虚拟内存地址。操作系统配合 CPU 的内存管理单元 (MMU) 为不同进程建立页表，页表记录虚拟地址到物理地址的映射关系。不同进程的虚拟内存空间的起始地址在逻辑上是一致的，所以进程通常认为自己拥有从 0 开始的全部内存空间。而 C 语言中的地址就是虚拟内存地址，打印一个指针的值：

int *p;
printf("%p", p);

输出的数值就是进程的某个虚拟地址。

2. 指针

C 语言的核心是指针。

指针本质上是一个存储内存地址的变量。让程序员能够直接读取或修改该地址上存储的数据。理解指针要明确 * 符号的两点意义：

  1. 指针声明
int *ptr;

C 风格的类型说明的设计哲学是一个变量的声明形式，与其在表达式中被使用时的形式相似。一般类型声明由类型说明符和声明符组成，这个例子中，* 是声明符 *ptr的一部分，告诉编译器这个变量存放的是地址，所以我认为 * 应当靠近指针变量名，无论何时不应该与类型说明符写在一起。 指针运算依赖于类型说明符，例如ptr + 1，编译器知道这是一个int型指针，所以地址要偏移 4 字节。

  1. 解引用
int a = *ptr;

参考 Java 中的引用变量，引用变量是”引用“某个地址上的数值，存储的是 JVM 中的抽象地址；指针则是存储一个地址，同样是”引用“这个地址上的数值。我们知道ptr本身的值是一个地址，需要 * 解出地址指向的数值，就是”解引用“。

3. 数组

要明确 C 语言可以直接操作地址，所以一切都变得很底层。

int nums[] = {1, 2, 3};

这条语句里nums[]是声明符，声明一个名为nums的数组。 nums的身份很复杂：

  • 某些情况下例如sizeof(nums)或&nums，代表整个数组，代表这块连续的内存区域；
  • 在表达式中隐式转换为指向首元素的指针，数值相当于&nums[0]；
  • []其实是通过指针运算解引用，``nums[i]等同于*(nums + i)`。
