1. 地址

C 语言的编译器会把.c文件编译为机器码存入一个.exe文件，机器码可以对应汇编语言，是直接与 CPU 对话的语言。各类操作系统比如 Windows、Linux 会提供 API 支持 C 程序的执行，可以看作一个巨大的“运行环境”。对比 Java 的编译器会把.java文件编译为字节码存入一个.class文件，其实思路是相似的，字节码是与 JVM 对话的语言，JVM 就是一个虚拟机，模拟了操作系统和 CPU，它需要把字节码翻译成各种机器码与不同架构的 CPU 对话。JVM 是运行在系统上的，而 Java 程序是运行在 JVM 上的。

算法基础 9. 寻找环形链表入口

Semones published on 2022-10-03 included in 算法基础

推导出的数学公式可直接用代码暴力实现，不必考虑原理

1. 基本要求

  1. 一个链表内部可能存在环形结构，返回链表开始入环的第一个节点，如果无环返回 null。
  2. 索引从 0 开始，用 pos 表示尾结点连接到链表中的位置。
  3. 不允许修改链表。

2. 代码实现

/*
  将链接视为等长路程，设头结点到入口结点路程为 x，入口结点到相遇结点为 y，相遇结点到入口结点为 z。
  快指针的速度是慢指针的两倍，若两者在圈内同时起步到相遇，慢指针刚好走完一圈。
  而在题设中相遇时，快指针提前入环，慢指针必定不会走完一圈，所以慢指针走过的路程为 x + y。
  设快指针已绕环 n 圈，则走过的路程为 x + y + n * (y + z)。
  慢指针路程 * 2 = 快指针路程，化简后得 x = (n - 1) * (y + z) + z 且 n 大于等于 1。
  这个公式说明：从头结点到入口结点的路程 x，等于从相遇结点到入口结点的路程 z 加上 n 圈路程。
  所以让两个同速指针分别从头结点和相遇结点出发，会在入口结点相遇。
*/


public class DetectCycle {
	// 快慢指针
	public ListNode getNode(ListNode head) {
		ListNode fast = head;
		ListNode slow = head;
		// 记录有环的真假
		boolean hasCycle = false;

		// 判断是否有环，快指针每次移动两步，慢指针每次一步，总会相遇
		while (fast != null && fast.next != null) {
			fast = fast.next.next;
			slow = slow.next;
			if (fast == slow) {
				hasCycle = true;
                break;
			}
		}

		// 这个判断在无环时成立，也就是需要有环的相反
		if (!hasCycle) {
			return null;
		}

		// 将 slow 重新指向头，然后与 fast 同速移动，相遇点即为环入口
		slow = head;
		while (slow != fast) {
			slow = slow.next;
			fast = fast.next;
		}

		return slow;
	}
}

算法基础 7. 反转链表

Semones published on 2022-08-03 included in 算法基础

递归是自顶向下，逐层返回，出口是关键。

1. 基本要求

  1. 反转单链表。
  2. 不能创建新链表。

2. 代码实现

  1. 迭代
public class ReverseListIteration {
	// 迭代
	public ListNode getList(ListNode head) {
		ListNode pre = null;
		ListNode cur = head; // 当前结点
		ListNode nxt = null;

		// 修改每条链接的方向
		while (cur != null) {
			nxt = cur.next; // 先保存后继结点
			cur.next = pre; // 反转当前链表的 next
			pre = cur;
			cur = nxt; // 后移前驱和当前的指针，准备反转下一条链接
		}

		// 最终时 pre 停留在最后一个结点
		return pre;
	}
}
  1. 递归
/*
  自顶向下：调用自身，传入去除头结点的子链表，层层开辟递归栈。
  递归出口：当传入最后一个结点时，作为新头结点返回，开始自下而上处理子链表。
  回溯操作：把当前结点连接到传入的子链表的尾部。
  逐层返回：向上传递子链表。
*/


public class ReverseListRecursive {
	// 递归
	public ListNode getList(ListNode head) {
		if (head == null) return null;
		// 递归的出口
		if (head.next == null) return head;

		// newHead 接收处理好的子链表，指向最终的新头结点，也就是原链表的尾结点
		ListNode newHead = getList(head.next);
		// 当前层的结点的 next 指向的是传入的子链表的尾结点
		// 把当前层结点赋给尾结点的 next，即可把当前结点连接到尾部
		head.next.next = head;
		head.next = null; // 尾结点指向 null

		// 把子链表传递上去
		return newHead;
	}
}

算法基础 8. 移除链表元素

Semones published on 2022-07-05 included in 算法基础

虚拟头结点可以记录头结点，方便指针移动。

1. 基本要求

  1. 删除链表等价于给定值的结点。
  2. 返回新的链表头结点。

2. 代码实现

  1. 迭代
public class RemoveElementsIteration {
	// 迭代
	public ListNode getList(ListNode head, int val) {
		// 虚拟头结点记录新的头结点
		ListNode dummy = new ListNode();
		dummy.next = head;

		// cur 指向虚拟头结点，所以 cur.next 是第一个真实结点
		ListNode cur = dummy;
		while (cur.next != null) {
			if (cur.next.val == val) {
				// 此时判断 cur.next 是要删除的结点
				// 用下一个结点的地址覆盖 cur.next 这个变量，达到删除 cur.next 的目的
				cur.next = cur.next.next;
			} else {
				cur = cur.next;
			}
		}

	// 如果原头结点被删，只能用 dummy.next 来表示新的头结点
	return dummy.next;
	}
}
  1. 递归
/*
  自顶向下：调用自身，传入去除头结点的子链表，层层开辟递归栈。
  递归出口：当传入的值为空时，返回这个值，开始自下而上处理子链表。
  回溯操作：比较处理结点元素。
  逐层返回：将处理好的子链表传给上一层结点的 next，以此连接起来。
*/


public class RemoveElementsRecursive {
	// 递归
	public ListNode getList(ListNode head, int val) {
		// 递归的出口
        if (head == null) {
            return head;
        }

        head.next = getList(head.next, val);
        if (head.val == val) {
            // 跳过此结点，把下层传上来的值直接传给上层
            return head.next;
        }

        // 把此结点的值传给上层
        return head;
	}
}
